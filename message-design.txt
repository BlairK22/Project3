Request Pipe Format (Client -> Server):
----------------------------------------
Format: "book:chapter:verse"
Example: "43:3:16"

The request is a string containing three integers separated by colons,
representing the book number, chapter number, and verse number.
This matches the format used by the Ref parse constructor.

Reply Pipe Format (Server -> Client):
--------------------------------------
Format: "status:verse_text"
Example: "0:John 3:16 For God so loved the world..."

The first character is the status code (0=SUCCESS, 1=NO_BOOK,
2=NO_CHAPTER, 3=NO_VERSE, 4=OTHER), followed by a colon,
followed by the verse text (if successful) or an error message.

The reply is terminated by a "$end" message to signal the client
that the server has finished sending data for this request.
