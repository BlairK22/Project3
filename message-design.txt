Request Pipe Format (Client -> Server):



Format: "book:chapter:verse:numVerses"

Example: "43:3:16:1" for John 3:16, single verse

Example: "40:5:43:10" for Matthew 5:43, 10 verses

The request is a string of integers separated by colons:

- book: book number (1-66)

- chapter: chapter number

- verse: starting verse number

- numVerses: how many consecutive verses to retrieve

Reply Pipe Format (Server -> Client):



Format: "status:BookName C:V verse_text"

Example: "0:John 3:16 For God so loved the world..."

The status code values are:

- 0 = SUCCESS

- 1 = NO_BOOK

- 2 = NO_CHAPTER

- 3 = NO_VERSE

For multi-verse requests, the server sends one reply message

per verse. Each reply contains the status and full verse data.

The response is terminated by a "$end" message to signal

the client that the server has finished sending.
